一、业务场景
读取日志文件 app.log，统计每种日志级别的出现次数，结果写入 result.txt。
日志格式示例：
[INFO] 2025-01-01 10:00:00 User login success
[ERROR] 2025-01-01 10:01:00 NullPointerException
[WARN] 2025-01-01 10:02:00 Memory usage high

二、功能需求

2.1 核心功能
多线程并发解析：使用线程池按行并发解析日志
统计日志级别数量：提取 []中的级别（INFO / ERROR / WARN），统计出现次数
结果写入文件：将统计结果输出到 result.txt
反射创建分析器：通过类名动态加载并创建分析器对象
Lambda 简化操作：用于集合统计和排序
全局异常处理：文件不存在、反射失败、IO 错误等均有友好提示

2.2 功能详细说明
① 多线程并发解析
使用 ExecutorService创建固定大小线程池
每行日志作为一个任务提交到线程池
主线程通过 shutdown()+ awaitTermination()等待所有任务完成
② 统计日志级别
从日志行中提取 []内的内容作为日志级别
使用 ConcurrentHashMap保证并发安全
统计结果示例：
INFO : 2
ERROR: 1
WARN : 1
③ 结果写入文件
输出文件：result.txt
使用 BufferedWriter+ try-with-resources
UTF-8 编码，覆盖写入
④ 反射动态创建分析器
通过配置文件或硬编码类名，使用 Class.forName()加载类
使用 getDeclaredConstructor().newInstance()创建实例
面向接口编程：LogAnalyzer analyzer = ...
⑤ Lambda 表达式应用
至少使用一次 Lambda 或方法引用，例如：
ConcurrentHashMap::merge
Map.forEach((k, v) -> ...)
Stream.sorted(Comparator.comparingInt(...))

三、面向对象设计

3.1 类结构
com.example.log
├── LogRecord.java          // 日志实体（封装）
├── LogAnalyzer.java        // 抽象分析器（继承）
├── CountBasedAnalyzer.java // 具体实现（多态）
├── AnalyzerFactory.java    // 反射创建对象
├── LoggerAnalyzer.java     // 多线程调度
├── ResultWriter.java       // IO 写结果
└── Main.java               // 程序入口

3.2 三大特性体现

四、关键技术点要求

4.1 String 底层（必写注释）
在 LogRecord#getLevel()中必须说明：
JDK 8：char[]；JDK 9+：byte[]+ coder
substring()返回新 String 对象，不修改原字符串
String 不可变性保证了多线程环境下的安全性

4.2 多线程与锁
使用 ConcurrentHashMap进行原子统计
或 synchronized修饰统计方法
或 ReentrantLock显式加锁
能说明锁的作用范围和选择理由

4.3 IO 流
读：BufferedReader
写：BufferedWriter
编码：UTF-8
资源关闭：try-with-resources

4.4 异常处理
场景
处理方式
文件不存在
提示"日志文件不存在"
反射失败
提示"分析器加载失败"
IO 异常
提示"文件读写失败"
线程中断
恢复中断标志

五、文件格式

输入：app.log
[INFO] 2025-01-01 10:00:00 User login success
[ERROR] 2025-01-01 10:01:00 NullPointerException
[WARN] 2025-01-01 10:02:00 Memory usage high
[INFO] 2025-01-01 10:03:00 User logout
输出：result.txt
INFO : 2
ERROR: 1
WARN : 1

六、运行流程
程序启动
通过 AnalyzerFactory（反射）创建分析器
创建线程池，按行读取 app.log
每行日志提交一个解析任务
分析器统计日志级别
主线程等待所有任务完成
将结果写入 result.txt
程序结束
